# Customer Churn Prediction Dashboard

An end-to-end machine learning project that predicts telecom customer churn using Python, Scikit-learn, and Streamlit.

Run the dashboard locally:


Then open your browser at `http://localhost:8501`.

## ML Pipeline

1. Generate a synthetic telecom churn dataset with 7,000 customers.
2. Perform EDA and save visual insights to `screenshots/`.
3. Engineer customer behavior and billing features.
4. Train Random Forest, Gradient Boosting, and Logistic Regression models.
5. Save the best model artifacts for the Streamlit dashboard.

## Models Trained

| Model | AUC | F1 | Accuracy |
|---|---:|---:|---:|
| Random Forest | 0.712 | 0.442 | 65.9% |
| Gradient Boosting | 0.704 | 0.456 | 65.0% |
| Logistic Regression | 0.731 | 0.485 | 65.1% |

Logistic Regression is selected as the best model because it has the strongest AUC and F1 score in this experiment.

## Key Insights

- Month-to-month contracts have the highest churn rate.
- Fiber optic internet customers churn more than DSL users.
- New customers with tenure under 12 months are the most at-risk segment.
- Higher monthly charges are positively correlated with churn.
- Electronic check payment customers show higher churn behavior.
- Long-term customers with tenure above 48 months rarely churn.
